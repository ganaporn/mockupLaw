# Mapping สถานะ กับ Role — ระบบติดตามคดี

> วันที่จัดทำ: 2026-06-28

---

## ภาพรวม Flow คดี

```
คำขอสร้างคดี (Request)
    ↓ อนุมัติโดย กลุ่มงานคดี
    ↓ assign ไปยัง →  เตรียมคดี | งานคดี | บังคับคดี
    ↓
┌─────────────────────────────────────────────┐
│  STAGE 1: กลุ่มงานคดี (Case Group)           │
│  STAGE 2: เตรียมคดี (Prepare)               │
│  STAGE 3: งานคดี (Litigation)               │
│  STAGE 4: บังคับคดี (Enforcement)           │
└─────────────────────────────────────────────┘
```

### ประเภทคำขอ → Stage เริ่มต้น

| ประเภทคำขอ | Stage เริ่มต้น |
|---|---|
| อุบัติเหตุ | เตรียมคดี |
| รถร่วม (บริหารสัญญา) | เตรียมคดี |
| นิติกรรม | เตรียมคดี |
| กลุ่มงานคดี (เป็นจำเลย) | งานคดี |

---

## STAGE 1 — กลุ่มงานคดี

**ผู้รับผิดชอบ:** กลุ่มงานคดี

| | รายละเอียด |
|---|---|
| **สถานะที่เห็นได้** | เสร็จสิ้น / ส่งงานคดี / ส่งงานบังคับคดี / ส่งเรื่องคืนงานเตรียมคดี / ส่งเรื่องกลับงานคดี |
| **สถานะที่เลือกได้** | รอดำเนินการ / ปิดคดี |
| **Assign ไปยัง** (กรณีรอดำเนินการ) | เตรียมคดี / งานคดี / บังคับคดี |
| **หมายเหตุ** | กรณี **ปิดคดี** = งานคดีเสร็จสิ้น ไม่ต้อง assign ให้ใคร |

## STAGE 2 — เตรียมคดี

| | รายละเอียด |

## STAGE 4 — บังคับคดี

### Role: ธุรการ

| | รายละเอียด |
|---|---|
| **สถานะที่เห็นได้** | รอดำเนินการ / นิติกรดำเนินการ (สถานะย่อย: ยื่นฟ้องลูกหนี้เป็นคดีล้มละลาย) |
| **สถานะที่เลือกได้** | รับเอกสาร / เสร็จสิ้น / ส่งเรื่องกลับงานคดี |
| **Action พิเศษ** | กรณีสถานะ **รับเอกสาร** → เลือกนิติกร → บันทึก → ระบบเปลี่ยนสถานะเป็น **นิติกรดำเนินการ** |

### Role: นิติกร

| | รายละเอียด |
|---|---|
| **สถานะที่เห็นได้** | นิติกรดำเนินการ |
| **สถานะที่เลือกได้** | สอบทาน |
| **สถานะย่อย** (ขณะนิติกรดำเนินการ) | ตรวจสอบเอกสาร / สืบทรัพย์ / ยึดทรัพย์-ขายทอดตลาด / ประนอมหนี้ / ยื่นฟ้องลูกหนี้เป็นคดีล้มละลาย / ยื่นคำขอรับชำระหนี้ / เสนอขออนุมัติงดบังคับคดี (ครบอายุความ 10 ปี) / เสร็จสิ้น |
| **สถานะย่อยของ เสร็จสิ้น** | อนุมัติงดบังคับคดีเพื่อยุติเรื่อง / รับชำระหนี้ |

### Role: หัวหน้างาน

|---|---|
| **สถานะที่เห็นได้** | สอบทาน |
| **สถานะที่เลือกได้** | ปิดเรื่อง |
| **สถานะย่อยของ ปิดเรื่อง** | ชำระหนี้ครบ / จำหน่ายหนี้ |

## โครงสร้างหน้า Case Tracking (case-tracking.html)

### 4 Tabs
| Tab | ชื่อ | เนื้อหา |
|---|---|---|
| Tab 1 | 📋 ข้อมูลทั่วไป | ข้อมูลพื้นฐานคดี (ประเภท, ชื่อเรื่อง, หน่วยงาน, มูลค่าเรียกร้อง, วันที่เปิดคดี) |
| Tab 2 | 📎 แนบไฟล์ | รายการเอกสารแนบ (ตาราง + เพิ่มเอกสาร) |
| Tab 3 | ⚖️ ข้อมูลคดีในศาล | เรื่อง, คดีหมายเลขดำ, คดีหมายเลขแดง, ศาล, ผู้ฟ้องคดี, ผู้ถูกฟ้องคดี |
| Tab 4 | 🔄 สถานะการทำงาน | บันทึกสถานะตาม Role + รายละเอียด + แนบไฟล์ + Timeline History |

### Tab 4 — Logic บันทึกสถานะตาม Role

```
Role: ธุรการ
  → แสดงสถานะที่เลือกได้ตาม Stage ปัจจุบัน
  → กรณีสถานะ "รับเอกสาร" หรือ "รับหมาย" → แสดง dropdown เลือกนิติกร
  → กด "บันทึก" → สถานะเปลี่ยนเป็น "นิติกรดำเนินการ" พร้อม notify นิติกร

Role: นิติกร
  → เห็นเฉพาะงานที่ได้รับ assign
  → เลือกสถานะย่อย (action ที่ดำเนินการ)
  → กด "ส่งสอบทาน" → สถานะเปลี่ยนเป็น "สอบทาน" พร้อม notify หัวหน้า

Role: หัวหน้างาน
  → เห็นเฉพาะงานสถานะ "สอบทาน"
  → เลือกผลลัพธ์ (อนุมัติ / ตีกลับ / ปิดเรื่อง / ส่งต่อ stage อื่น)
  → กด "บันทึก" → ระบบอัปเดต stage ใหม่
```
